# 越野跑比赛时间预测系统 V0.1 开发需求文档

## 1. 项目目标

开发一个本地运行的 Python 越野跑比赛时间预测工具。

核心功能：用户上传多个历史 `.fit` 运动文件和一个比赛路线 `.gpx` 文件，系统分析用户历史运动表现，建立个人越野能力模型，然后根据比赛路线特征预测比赛完成时间。

第一阶段不使用数据库，不做前后端分离，全部使用 Python 本地运行，目标是验证：

> 根据个人历史 FIT 数据，是否可以预测未来越野比赛完成时间。

## 4. 第一阶段功能：FIT 文件解析


```text

```

示例：

| time | distance | altitude | HR |
| --- | ---: | ---: | ---: |
| 08:00:01 | 0 | 120 | 90 |
| 08:00:02 | 5 | 121 | 92 |

## 5. 第二阶段功能：运动数据分析

针对每个 FIT 文件计算以下基础指标：

```text
distance_km

elevation_loss
avg_hr
max_hr
avg_cadence
```

## 6. 个人能力模型

生成 `runner_profile.json`，示例：

```json
{
  "flat": {
    "aerobic_pace": 300,
    "threshold_pace": 260
  },
  "uphill": {
    "5_percent": 800,
    "10_percent": 650,
    "15_percent": 450
  },
  "downhill": {
    "-5_percent": 1200,
    "-15_percent": 900
  },
  "fatigue": {
    "3h": 0.95,
    "5h": 0.85,
    "8h": 0.75
  }
}
```

## 7. 能力计算规则

### 7.1 平路能力

筛选坡度为 `-2%～+2%` 的数据，按以下公式计算速度：

```text
速度 = 距离 / 时间
```

按心率区分：

- 有氧能力：HR 140～155
- 阈值能力：HR 160～175

### 7.2 上坡能力

核心指标为 VAM：

```text
VAM = 爬升高度（m）/ 时间（h）
```

根据坡度分类：

| 坡度 | 类型 |
| --- | --- |
| 5%～10% | 缓坡 |
| 10%～15% | 中坡 |
| 15%以上 | 陡坡 |

最终生成“坡度 → VAM”曲线。

### 7.3 下坡能力

计算下降速度：

```text
下降速度 = 下降高度 / 时间
```

根据 `-5%`、`-10%`、`-15%` 等坡度建立下降能力模型。

### 7.4 长距离疲劳模型

分析长距离 FIT 中同一个人的速度变化，例如：

```text
0～3 小时：100%
3～5 小时：90%
5 小时以后：80%
```

最终生成 `fatigue_factor`。

## 8. GPX 路线解析

输入 `race.gpx`，解析每个轨迹点的以下字段：

```text
latitude
longitude
elevation
```

计算：

- 距离
- 爬升
- 下降
- 坡度

转换成比赛 Segment，示例：

```json
{
  "distance": 500,
  "gain": 80,
  "loss": 0,
  "grade": 16
}
```

建议每 500 米作为一个预测单元。

## 9. 比赛预测算法

输入 `runner_profile.json` 和 `race_segments`，逐段进行预测。

### 9.1 上坡

条件：

```text
grade > 5%
```

计算：

```text
time = gain / VAM
```

### 9.2 下坡

条件：

```text
grade < -5%
```

计算：

```text
time = distance / downhill_speed
```

### 9.3 平路

条件：

```text
-5% <= grade <= 5%
```

计算：

```text
time = distance / flat_speed
```

### 9.4 疲劳修正

根据比赛累计时间调整：

```text
segment_time *= fatigue_factor
```

## 11. 开发顺序

严格按照以下顺序：

1. 完成 FIT 文件解析，目标是成功读取自己的 FIT 文件。
2. 完成单个 FIT 基础分析。
3. 完成多个 FIT 汇总并生成个人能力模型。
4. 完成 GPX 文件解析。
5. 完成比赛时间预测。
6. 增加 Streamlit UI。

## 12. 第一版验收标准

使用自己的历史数据测试。

输入：

- 多个训练 FIT
- 已完成比赛 FIT

输出：预测时间。

第一版预测误差达到 `10%～15%`，即可认为模型有效。
